## Why I Built This

Planning a night out is fragmented. People check google maps, reviews, social media, group chats, and venue pages just to answer simple questions:

- What are some good clubs in this city?
- Where is actually worth going tonight?
- Is it busy?
- What is the vibe?
- Can I wear jeans?
- How long is the wait?
- How much is cover?
- What kind of music do they play?
- Can my group decide quickly?

LIVE centralizes that decision making flow into one nightlife discovery experience.

## Core Features

| Feature | Description |
|---|---|
| Venue discovery | Browse nightlife venues by location, popularity, and vibe |
| Live activity signals | Show crowd/activity indicators to help users decide |

|---|---|

| Styling | Tailwind CSS |
| Backend | Supabase + Supabase Edge Functions |
| Database | PostgreSQL / AWS |
| Maps | Google Maps API / Mapbox |
| Design | Figma | Google Stich | Claude | Nano Banana

## Technical Highlights

- Built reusable UI components from Figma mockups
- Structured the app around fast venue comparison
- Designed a responsive landing page to support the product narrative
- Created a consistent visual identity across brand, app, and marketing surfaces
- Created a agent orchestration framework to build end to end designs
